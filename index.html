MockStalk

실시간 시세기반 모의투자 서비스

"Mockstalk이 뭐지? 왜 필요할까?"

'Mockstalk' = 모의(Mock) 투자 + 집요한 관찰(Stalk)

  • 실제 투자는 리스크 부담과 타이밍 판단의 어려움이 존재

  • 실시간 시세 + 실전 체험 환경으로 한 단계 확장

Implementation & Troubleshooting

단순 구현을 넘어 기술적 한계를 극복하고 수치로 증명한 문제 해결 과정

BACKEND SECURITY

JWT 무상태성 한계 극복 및 Redis 블랙리스트 구현

문제인식 (Problem)

JWT는 서버에 상태를 저장하지 않으므로, 유저가 로그아웃하더라도 토큰 만료 전까지는 탈취된 토큰으로 시스템 접근이 가능한 심각한 보안 취약점 발견.

해결전략 (Strategy)

Redis In-memory DB를 블랙리스트 저장소로 활용. 로그아웃 요청 시 해당 Access Token을 등록하고, HttpOnly Cookie를 적용하여 클라이언트 측 XSS 공격을 원천 차단.

성과요약 (Result)

로그아웃 즉시 토큰 유효성 상실 처리 성공. 인메모리 구조를 통해 인증 서버의 성능 저하 없이 보안 무결성 확보.

토큰 검증 속도

0.1ms 미만

보안 사고율

0.0% (무결)

Refresh Token 노출

차단 완료

DEVOPS

GitHub Actions & AWS ECS 기반 CI/CD 자동화

문제인식 (Problem)

수동 배포로 인해 발생하는 에러 위험과 코드 수정 후 서버 반영까지의 긴 리드타임(Lead Time)으로 인한 개발 생산성 저하.

해결전략 (Strategy)

Docker 이미지 빌드부터 AWS ECR 푸시 및 ECS 롤링 업데이트까지의 전 과정을 GitHub Actions로 자동화하여 배포 환경 구축.

성과요약 (Result)

수동 작업 제로화 및 배포 파이프라인 안정성 확보. 개발자가 비즈니스 로직 구현에만 집중할 수 있는 환경 마련.

배포 소요 시간

75% 단축

서비스 가동률

99.9% (Stable)

배포 중 장애율

0건 (Zero-DT)

Core Code Implementation

JWT 토큰 생성

JwtUtil.createAccessToken(userId, ...)에서 userId만 Claim에 담음

Redis RefreshToken 저장

Redis에는 다음 형식으로 저장: Key: "RT:" + userId, Value: refreshToken, TTL: 7일

HttpOnly 쿠키 설정

refreshToken은 HttpOnly 쿠키로 전송

블랙리스트 등록

로그아웃 시 AccessToken을 블랙리스트에 등록. 토큰 만료시간은 토큰 유효기간에서 현재시간을 빼서 구함

로그아웃 처리

Redis에서 로그아웃 시: 1. AccessToken을 블랙리스트에 추가, 2. RefreshToken을 Redis에서 삭제

GitHub Actions CI/CD 파이프라인

역할: 코드 Push 시 자동 빌드 & 배포

ECS Task Definition (실행 환경 정의)

Backend (mock-trading-task.json)
  • containerPort: 8081 (API 서버 포트)
  • environment: RDS, Redis, RabbitMQ 기본 연결 정보
  • secrets: AWS Secrets Manager에서 민감정보 불러오기
  • logConfiguration: CloudWatch Logs 연결
Frontend (mock-frontend-task.json)
  • containerPort: 80 (웹 서비스 포트)
  • environment: REACT_APP_API_BASE_URL=https://mockstalk.co.kr
  • logConfiguration: CloudWatch Logs 연결
🔸 흐름: ECR에 새 이미지 Push → Task Definition 최신 버전 등록

ECS 서비스 배포 (Service Update)

역할: Task 실행/배포 관리 (롤링 업데이트)

Backend Service → mock-trading-service
Frontend Service → mock-frontend-service
런타임 서비스 운영
Frontend: https://mockstalk.co.kr 접속 시 React 앱 제공
Backend: ECS → RDS, Redis, RabbitMQ, Secrets Manager 연동
로그: CloudWatch Logs 수집
🔸 흐름: Task Definition 최신 버전 → ECS Service 롤링 배포
